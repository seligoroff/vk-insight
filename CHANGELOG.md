# Changelog

Все значимые изменения в проекте документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и проект следует [Semantic Versioning](https://semver.org/lang/ru/).

## [0.8.0] - 2026-03-01

### Добавлено
- **Новая команда `vk:posts-get-all`** (ранее `vk:posts-get-all-groups`)
  - Выполняет `vk:posts-get --db --clear` для каждой группы из `resources/vk-groups.csv`
  - Автоматически обрабатывает все группы из списка
  - Показывает progress bar с информацией о текущей группе
  - Поддерживает настройку задержки между запросами
  - Выводит итоговую статистику по обработанным группам и сохраненным постам
- **Новая команда `vk:likers-core`** - вычисление ядра лайкнувших пост по дружеским связям
  - Получение списка лайкнувших через `likes.getList`
  - Подсчет внутренних связей среди лайкнувших через `friends.get`
  - Порог включения в ядро через параметр `--k`
  - Защита от перегрузки API через `--max-users` и `--delay`
  - Поддержка форматов `table`, `json`, `csv`, `markdown` и сохранение в файл
  - Расширенный вывод профилей (`name`, `screen_name`, `profile_url`)
- **Новые VK API сервисы**
  - `VkLikesService` для работы с лайками
  - `VkFriendsService` для работы с друзьями
  - `VkUsersService` для получения профилей пользователей

### Изменено
- **Улучшена команда `vk:stats-get`**
  - Добавлен формат `markdown`
  - Добавлена опция `--verbose-raw` для вывода сырого ответа VK API
  - Исправлено сохранение в файл при `--format=table` (теперь сохраняется текстовый отчет)
  - Добавлено автоопределение формата по расширению файла (`.json`, `.csv`, `.md`, `.markdown`)
  - Добавлена фильтрация периодов ответа строго по диапазону `--from/--to`
  - Исправлен парсинг `--to`: теперь используется конец дня (`23:59:59`)
- **Улучшена команда `vk:token-check`**
  - Добавлена проверка права `stats` в битовой маске прав доступа
- **Расширен `VkSdkAdapter`**
  - Добавлены методы `likes()` и `friends()` для новых команд

### Документация
- Обновлены `docs/commands.md`, `docs/examples.md`, `docs/configuration.md`, `docs/troubleshooting.md`
  - Добавлены инструкции по `vk:likers-core`
  - Добавлены актуальные примеры `vk:stats-get`
  - Синхронизированы требования по правам токена (`friends`, `stats`)
  - Добавлены рекомендации по диагностике нулевой статистики в `vk:stats-get`

## [0.6.0] - 2026-01-06

### Изменено
- **Миграция на официальный VK PHP SDK (`vkcom/vk-php-sdk`)**
  - Все сервисы работы с VK API теперь используют официальную библиотеку VK
  - `VkGroupService`, `VkWallService`, `VkPhotoService`, `VkAudioService`, `VkApiTestService` мигрированы на SDK
  - Создан адаптер `VkSdkAdapter` для удобной работы с SDK
  - Удален старый базовый класс `VkApiClient`
  - Все команды работают без изменений благодаря обратной совместимости интерфейсов сервисов
  - Преимущества: типобезопасность, автодополнение, официальная поддержка, автоматические обновления API

### Добавлено
- **Улучшение команды `vk:check`**
  - Добавлена колонка "Время с публикации" во все форматы вывода
  - Формат отображения времени: "X дн Y ч" (комбинированный)
  - Автоматическое переключение между днями/часами/минутами в зависимости от интервала
  - Добавлено поле `post_date` в таблицу `vk_check_cache` для хранения времени публикации поста

### Удалено
- Старый базовый класс `VkApiClient` (заменен на использование SDK)
- Feature-тесты, использовавшие устаревшие моки `Http::fake()` (требуют переработки под SDK)

### Технические детали
- Все сервисы поддерживают dependency injection через метод `setAdapter()` для тестирования
- Unit-тесты обновлены для использования Mockery вместо `Http::fake()`
- Рекурсивное преобразование массивов в объекты для обратной совместимости

## [0.2.4] - 2026-01-03

### Изменено
- Опция `--clear` в команде `vk:posts-get` теперь очищает только посты указанного владельца (`--owner`), а не всю таблицу
  - Это позволяет работать с несколькими группами одновременно, очищая данные только для конкретного владельца
  - Посты других владельцев остаются нетронутыми при использовании `--clear`

### Добавлено
- Unit тесты для опций `--clear` и `--update` в команде `vk:posts-get`
  - Тест `test_clear_removes_only_specified_owner_posts` - проверяет, что очистка работает только для указанного владельца
  - Тест `test_update_updates_existing_posts` - проверяет обновление существующих записей
  - Тест `test_update_adds_new_and_updates_existing_posts` - проверяет одновременное обновление и добавление постов
- Примеры использования в документации (`docs/examples.md`)
  - Примеры работы с опциями `--update` и `--clear`
  - Новый раздел "Работа с несколькими группами одновременно" с практическими примерами

### Исправлено
- Исправлена логика очистки базы данных: `clearDatabase()` теперь вызывается в начале метода `handle()` перед получением постов из API
