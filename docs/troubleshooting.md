# Решение проблем

## Ошибки подключения к базе данных

### "SQLSTATE[HY000] [2002] Connection refused" или "Access denied"
- Проверьте настройки подключения к MySQL в `.env`:
  - `DB_HOST` - должен быть правильный хост (обычно `127.0.0.1` или `localhost`)
  - `DB_PORT` - должен быть правильный порт (обычно `3306`)
  - `DB_DATABASE` - база данных должна существовать
  - `DB_USERNAME` и `DB_PASSWORD` - учетные данные должны быть правильными
- Убедитесь, что MySQL сервер запущен: `sudo systemctl status mysql` или `mysqladmin ping`
- Проверьте, что база данных создана: `mysql -u root -p -e "SHOW DATABASES;"`
- Убедитесь, что у пользователя есть права доступа к базе данных

### "SQLSTATE[HY000]: General error: 10 disk I/O error"
- Эта ошибка характерна для SQLite при больших объемах данных
- Решение: используйте MySQL (приложение по умолчанию настроено на MySQL)
- Проверьте, что в `.env` указано `DB_CONNECTION=mysql`, а не `sqlite`

### "Table doesn't exist" или "Base table or view not found"
- Запустите миграции: `php artisan migrate`
- Проверьте, что все миграции выполнены: `php artisan migrate:status`

## Ошибка "Invalid token"
- Проверьте правильность токена в `.env`
- Убедитесь, что токен не истек
- Проверьте права доступа токена

## Ошибка "Access denied"
- Убедитесь, что токен имеет необходимые права (`wall`, `groups`, `audio`, `friends`, `stats`)
- Проверьте, что группа/пользователь доступны для токена

## `vk:stats-get` возвращает нули или пустую демографию
- Проверьте, что используется **user token** с правом `stats` (а не только group token)
- Убедитесь, что пользователь токена — администратор сообщества
- Попробуйте включить `--extended` и посмотреть сырой ответ: `--verbose-raw --format=json`
- Для длинных периодов используйте `--interval=month` или `--interval=year`
- Нулевая демография возможна при низкой активности/недостаточной выборке

## Ошибка "Too many requests"
- Увеличьте задержку между запросами (`--delay`)
- Подождите некоторое время перед повторным запуском

## Файл `vk-groups.csv` не найден
- Создайте файл `resources/vk-groups.csv`
- Добавьте в него URL групп, по одной на строку
- Или используйте `make vk-groups-file` для автоматического создания

## Ограничения VK API

VK API имеет ограничения на количество запросов:
- **Rate limiting:** Инструмент автоматически добавляет задержки между запросами
- **Лимиты:** При превышении лимитов API вернет ошибку - нужно подождать

Рекомендуется:
- Использовать задержки между запросами (параметр `--delay`)
- Не запускать несколько команд одновременно
- Использовать кеш (`--cached`) для повторного просмотра результатов

## Поддержка

При возникновении проблем:
1. Проверьте настройки в `.env`
2. Убедитесь, что токен VK API валиден
3. Проверьте логи Laravel в `storage/logs/`
