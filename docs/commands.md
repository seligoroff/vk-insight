# Команды

## `vk:check` - Проверка последних постов в группах

Проверяет последние посты во всех группах из списка `vk-groups.csv` и выводит таблицу с метриками (лайки, репосты, количество подписчиков).

**Использование:**
```bash
# Базовое использование
php artisan vk:check

# Использование кеша
php artisan vk:check --cached

# С задержкой между запросами
php artisan vk:check --delay=1

# Экспорт в JSON
php artisan vk:check --format=json --output=check_results.json

# Экспорт в CSV
php artisan vk:check --format=csv --output=check_results.csv

# Автоматическое определение формата по расширению файла
php artisan vk:check --output=results.json  # автоматически JSON
php artisan vk:check --output=results.csv  # автоматически CSV
php artisan vk:check --output=results.md  # автоматически Markdown

# Явное указание Markdown формата
php artisan vk:check --format=markdown --output=check_report.md
```

**Параметры:**
- `--cached` - использовать кеш из предыдущего запуска (не делать запросы к API)
- `--delay=0.3` - задержка между запросами в секундах (по умолчанию 0.3)
- `--format=table` - формат вывода: `table`, `json`, `csv`, `markdown` (по умолчанию `table`)
- `--output` - путь к файлу для сохранения результатов (опциональный)
  - При указании расширения `.json`, `.csv`, `.md` или `.markdown` формат определяется автоматически
  - Если формат `table` и расширение не указано, сохраняется в CSV
  - При указании расширения `.md` или `.markdown` автоматически используется Markdown формат с таблицами

**Что выводится:**
- Текст последнего поста с текстом
- Название группы
- ID группы
- Количество лайков
- Количество репостов
- Количество подписчиков
- Время с момента публикации поста (формат: "X дн Y ч", "X дн", "Y ч", "X мин" или "только что")

**Примечание:** Результаты кешируются в таблице `vk_check_cache` базы данных для быстрого повторного просмотра.

## `vk:candidate` - Поиск кандидатов для републикации

Ищет посты со стены указанной группы/пользователя, которые можно использовать для републикации.

**Использование:**
```bash
php artisan vk:candidate --owner=-12345678
php artisan vk:candidate --owner=-12345678 --noreposts
```

**Параметры:**
- `--owner` - ID владельца стены (обязательный). Для групп используйте отрицательное число (например, `-12345678`)
- `--noreposts` - исключить посты, которые уже были отрепощены

**Примечание:** Команда проходит по всем постам стены, поэтому может занять время. Между постами есть задержка 5 секунд.

## `vk:posts-get-all` - Получение постов для всех групп из vk-groups.csv

Выполняет `vk:posts-get --db --clear` для каждой группы из файла `resources/vk-groups.csv`. Удобно для массового обновления данных всех групп за раз.

**Использование:**
```bash
# Получить посты за последний месяц для всех групп
php artisan vk:posts-get-all --from="last month"

# Получить посты за конкретный период
php artisan vk:posts-get-all --from=2024-01-01 --to=2024-01-31

# С увеличенной задержкой между запросами (для избежания rate limiting)
php artisan vk:posts-get-all --from=2024-01-01 --delay=0.5

```

**Параметры:**
- `--from` - Дата начала периода (обязательный). Форматы: `YYYY-MM-DD`, `YYYY-MM-DD HH:MM:SS`, относительные даты (`today`, `yesterday`, `last week`, `last month`)
- `--to` - Дата окончания периода (опциональный, по умолчанию текущая дата)
- `--delay=0.3` - Задержка между запросами к группам в секундах (по умолчанию 0.3)

**Что делает команда:**
1. Читает список групп из `resources/vk-groups.csv`
2. Для каждой группы:
   - Резолвит название группы в ID через VK API
   - Очищает старые посты этой группы из БД (аналог `--clear`)
   - Получает все посты за указанный период
   - Сохраняет посты в таблицу `vk_posts` (аналог `--db`)
3. Выводит итоговую статистику: количество обработанных групп, сохраненных постов, ошибок

**Особенности:**
- Автоматически пропускает группы, для которых не удалось получить посты
- Обрабатывает ошибки для каждой группы независимо (ошибка в одной группе не останавливает обработку остальных)
- Показывает progress bar с информацией о текущей группе
- Добавляет задержки между запросами для соблюдения rate limits VK API

**Пример вывода:**
```
Обработка 21 групп...
Период: с 2024-01-01 00:00:00 по 2024-01-31 23:59:59

 21/21 [████████████████████████████] 100% ✓ Сохранено 45 постов: groupname

=== Результаты ===
Обработано групп: 21
Всего сохранено постов: 523
```

**Примечания:**
- Команда использует ту же логику, что и `vk:posts-get --db --clear`, но применяет её ко всем группам автоматически
- Рекомендуется использовать задержку `--delay=0.5` или больше для больших списков групп
- Для каждой группы сначала удаляются все старые посты, затем загружаются новые за указанный период

**Сокращение:** Команда ранее называлась `vk:posts-get-all-groups`, теперь переименована в `vk:posts-get-all` для краткости

## `vk:posts-get` - Получение постов за период

Получает посты группы/пользователя за указанный временной период с возможностью фильтрации и экспорта данных.

**Использование:**
```bash
# Базовое использование
php artisan vk:posts-get --owner=-12345678 --from=2024-01-01

# С указанием конечной даты
php artisan vk:posts-get --owner=-12345678 --from=2024-01-01 --to=2024-01-31

# Сохранение в файл
php artisan vk:posts-get --owner=-12345678 --from=2024-01-01 --format=json --output=posts.json

# Сохранение в БД (накопление данных, дубликаты пропускаются)
php artisan vk:posts-get --owner=-12345678 --from=2024-01-01 --db

# Сохранение в БД с обновлением существующих записей
php artisan vk:posts-get --owner=-12345678 --from=2024-01-01 --db --update

# Очистка и полное обновление данных владельца
php artisan vk:posts-get --owner=-12345678 --from=2024-01-01 --db --clear
```

**Параметры:**
- `--owner` - ID владельца стены (обязательный, отрицательное число для групп)
- `--from` - Дата начала периода (обязательный). Форматы: `YYYY-MM-DD`, `YYYY-MM-DD HH:MM:SS`, относительные даты (`today`, `yesterday`, `last week`)
- `--to` - Дата окончания периода (опциональный, по умолчанию текущая дата)
- `--format` - Формат вывода: `table`, `json`, `csv` (по умолчанию `table`)
- `--output` - Путь к файлу для сохранения результатов (опциональный, по умолчанию вывод в stdout)
- `--db` - Сохранить результаты в базу данных (вместо файла, без вывода в консоль)
- `--update` - Обновить существующие записи в БД (только с `--db`). Если запись существует, она будет обновлена актуальными данными (лайки, репосты, текст)
- `--clear` - Очистить все посты указанного владельца перед вставкой (только с `--db`). Удаляет только посты указанного `--owner`
- `--with-text-only` - Показывать только посты с текстом
- `--min-likes` - Минимальное количество лайков для фильтрации
- `--min-reposts` - Минимальное количество репостов для фильтрации

**Примечания:**
- При использовании опции `--db` данные сохраняются в таблицу `vk_posts` без вывода в консоль. Убедитесь, что миграции выполнены (`make migrate`).
- По умолчанию (без `--update`) дубликаты предотвращаются: если пост уже существует в БД, он пропускается. Это позволяет накапливать данные без обновления.
- С опцией `--update` существующие записи обновляются актуальными данными (лайки, репосты, комментарии, текст), новые посты добавляются.
- С опцией `--clear` перед вставкой удаляются все посты указанного владельца (`--owner`). Полезно для полного обновления данных владельца. Это позволяет работать с несколькими группами одновременно, очищая данные только для конкретного владельца.

## `vk:albums-get` - Получение списка фотоальбомов

Получает список фотоальбомов пользователя или сообщества VK с возможностью фильтрации и экспорта данных.

**Использование:**
```bash
php artisan vk:albums-get --owner=-12345678
php artisan vk:albums-get --owner=-12345678 --need-system
php artisan vk:albums-get --owner=-12345678 --format=json --output=albums.json
php artisan vk:albums-get --owner=-12345678 --min-size=10
php artisan vk:albums-get --owner=-12345678 --need-covers --format=csv --output=albums.csv
```

**Параметры:**
- `--owner` - ID владельца альбомов (обязательный, отрицательное число для групп)
- `--format` - Формат вывода: `table`, `json`, `csv` (по умолчанию `table`)
- `--output` - Путь к файлу для сохранения результатов (опциональный, по умолчанию вывод в stdout)
- `--need-system` - Включить системные альбомы (с отрицательными ID)
- `--need-covers` - Получить обложки альбомов (thumb_src)
- `--min-size` - Минимальное количество фотографий в альбоме для фильтрации

**Примечание:** Системные альбомы (например, "Фотографии со мной") имеют отрицательные ID и не содержат полей `description`, `created`, `updated`. При использовании опции `--need-covers` в ответе API появляется поле `thumb_src` с URL обложки альбома.

## `vk:analytics` - Углубленный анализ эффективности постов

Проводит комплексный анализ эффективности постов группы/пользователя VK на основе данных из базы данных. Команда рассчитывает метрики вовлеченности, определяет лучшее время публикации, находит топ-посты и сравнивает периоды.

**📖 Подробная документация:** [Аналитика эффективности постов VK](analytics.md)

**Быстрый старт:**
```bash
# 1. Загрузить данные
php artisan vk:posts-get --owner=-12345678 --from=2024-01-01 --db

# 2. Запустить анализ
php artisan vk:analytics --owner=-12345678
```

**Основные параметры:**
- `--owner` - ID владельца стены (обязательный, отрицательное число для групп)
- `--period` - Период: `week`, `month`, `quarter`, `year` или `YYYY-MM-DD:YYYY-MM-DD` (по умолчанию `month`)
- `--best-time` - Показать лучшее время публикации
- `--compare=previous` - Сравнить с предыдущим периодом
- `--top=N` - Количество топ-постов (по умолчанию `10`)
- `--format` - Формат вывода: `table`, `json`, `csv` (по умолчанию `table`)
- `--output` - Путь к файлу для сохранения результатов
  - При указании расширения `.md` или `.markdown` автоматически используется Markdown формат

**Что анализируется:**
- Общая статистика (средние лайки, репосты, комментарии, ER)
- ER по дням недели
- Лучшее время публикации (с `--best-time`)
- Топ-посты по метрикам
- Сравнение периодов (с `--compare=previous`)

**Примечание:** Команда работает с данными из таблицы `vk_posts`. Перед использованием убедитесь, что данные загружены командой `vk:posts-get --db`. Подробные примеры и инструкции смотрите в [документации по аналитике](analytics.md).

## `vk:word` - Поиск постов по слову

Ищет посты, содержащие указанное слово, за указанный период и выводит статистику по найденным постам.

**Использование:**
```bash
# Поиск через API
php artisan vk:word концерт --owner=-12345678 --from=2024-01-01 --to=2024-01-31

# Поиск в базе данных (быстрее, не требует запросов к API)
php artisan vk:word выставка --owner=-12345678 --from=2024-01-01 --db

# Экспорт результатов в JSON
php artisan vk:word мероприятие --owner=-12345678 --from=2024-01-01 --format=json --output=word_search.json

# Экспорт в Markdown
php artisan vk:word конференция --owner=-12345678 --from=2024-01-01 --format=markdown --output=report.md

# Поиск в БД с экспортом в CSV
php artisan vk:word событие --owner=-12345678 --from=2024-01-01 --db --format=csv --output=results.csv
```

**Параметры:**
- `word` - Слово для поиска (обязательный аргумент)
- `--owner` - ID владельца (группа или пользователь), например `-12345678` (обязательный)
- `--from` - Дата начала периода (формат: `YYYY-MM-DD`, обязательный). Поддерживает относительные даты (`today`, `yesterday`, `last week`, `last month`)
- `--to` - Дата окончания периода (формат: `YYYY-MM-DD`, по умолчанию текущая дата)
- `--format=table` - Формат вывода: `table`, `json`, `csv`, `markdown` (по умолчанию `table`)
- `--output` - Путь к файлу для сохранения результатов (опциональный)
- `--db` - Поиск в таблице `vk_posts` вместо запросов к API (рекомендуется для больших объемов данных)

**Что выводится:**
- Количество найденных постов
- Сумма лайков
- Сумма комментариев
- Сумма репостов
- Средние значения (лайки, комментарии, репосты)

**Примечания:**
- Поиск регистронезависимый (не учитывает регистр букв)
- При использовании опции `--db` поиск происходит в таблице `vk_posts`, что намного быстрее и не требует запросов к API
- Для поиска через БД необходимо предварительно загрузить данные командой `vk:posts-get --db`
- Поиск работает по точному вхождению слова в текст поста

## `vk:stats-get` - Статистика сообщества через `stats.get`

Получает статистику сообщества VK (охват, посетители, активность, демография) за период.

**Использование:**
```bash
# Базовый запрос
php artisan vk:stats-get --group-id=12345678 --from=2025-01-01 --to=2025-01-31

# По месяцам, расширенная статистика
php artisan vk:stats-get --group-id=12345678 --from=2025-01-01 --to=2025-12-31 --interval=month --extended

# Сырой ответ VK API для диагностики
php artisan vk:stats-get --group-id=12345678 --from=2025-01-01 --to=2025-01-31 --extended --verbose-raw --format=json

# Экспорт в Markdown
php artisan vk:stats-get --group-id=12345678 --from=2025-01-01 --to=2025-01-31 --extended --output=reports/stats.md
```

**Параметры:**
- `--group-id` - ID группы (обязательный, положительное число)
- `--from` - Дата начала периода (обязательный, `YYYY-MM-DD`)
- `--to` - Дата окончания периода (опциональный, `YYYY-MM-DD`)
- `--interval=day` - Интервал группировки: `day`, `week`, `month`, `year`, `all`
- `--extended` - Расширенная статистика (демография и дополнительные срезы)
- `--verbose-raw` - Показать сырой JSON ответ VK API
- `--format=table` - Формат вывода: `table`, `json`, `csv`, `markdown`
- `--output` - Путь к файлу для сохранения результатов

**Примечания:**
- Для работы нужен user token с правом `stats`.
- Команда фильтрует интервалы строго в рамках `--from/--to`, если VK API вернет лишние периоды.

Вычисляет ядро лайкнувших пост на основе дружеских связей между лайкнувшими пользователями.

**Использование:**
```bash
# Базовый анализ
php artisan vk:likers-core --owner=-12345678 --post=12345

# Ядро с порогом 2 друзей среди лайкнувших
php artisan vk:likers-core --owner=-12345678 --post=12345 --k=2

# Экспорт в Markdown
php artisan vk:likers-core --owner=-12345678 --post=12345 --k=2 --output=reports/likers_core.md
```

**Параметры:**
- `--owner` - ID владельца стены (обязательный, отрицательное число для групп)
- `--post` - ID поста (обязательный)
- `--k=1` - Минимум друзей среди лайкнувших для включения в ядро
- `--max-users=300` - Максимум лайкнувших для анализа (защита от перегрузки API)
- `--delay=0.2` - Задержка между вызовами `friends.get` в секундах
- `--format=table` - Формат вывода: `table`, `json`, `csv`, `markdown`
- `--output` - Путь к файлу для сохранения результатов

**Что выводится:**
- Сводка: число проанализированных лайкнувших, размер ядра, ошибки чтения друзей
- Список пользователей ядра с полями:
  - `user_id`
  - `name`
  - `screen_name`
  - `friends_in_likers_count`
  - `friends_data_available`

**Примечание:** Для корректной работы нужен user token с правом `friends`.

## `vk:token-get-user` - Получение user access token

Интерактивная команда для получения ключа доступа пользователя (user access token) через OAuth 2.0 с использованием официальной библиотеки VK.

**Использование:**
```bash
# Интерактивный режим (команда запросит все параметры)
php artisan vk:token-get-user

# С указанием всех параметров
php artisan vk:token-get-user --client-id=12345678 --redirect-uri=https://oauth.vk.com/blank.html --scopes=wall,groups,photos,stats
```

**Параметры:**
- `--client-id` - ID приложения VK (Application ID) - обязательный
- `--redirect-uri` - URL для перенаправления после авторизации (например, `https://oauth.vk.com/blank.html`) - обязательный
- `--scopes` - Права доступа через запятую (по умолчанию: `wall,groups,photos,stats`)

**Поддерживаемые права доступа (scopes):**
- `wall` - Доступ к стенам
- `groups` - Доступ к группам
- `photos` - Доступ к фотографиям
- `video` - Доступ к видео
- `docs` - Доступ к документам
- `friends` - Доступ к друзьям
- `stats` - Доступ к статистике
- `pages` - Доступ к страницам
- `notes` - Доступ к заметкам
- `stories` - Доступ к историям
- `ads` - Доступ к рекламе
- `market` - Доступ к товарам
- `notifications` - Доступ к уведомлениям
- `status` - Доступ к статусу

**Как работает:**
1. Команда запрашивает параметры приложения (client_id, redirect_uri, scopes)
2. Генерирует URL для авторизации через VK ID API
3. Показывает URL, который нужно открыть в браузере
4. После авторизации пользователь вставляет URL с кодом из адресной строки
5. Команда получает токен и выводит его для добавления в `.env`

**Примечания:**
- Команда использует официальную библиотеку `vkcom/vk-php-sdk`
- Используется современный VK ID API с поддержкой PKCE для безопасности
- Токен можно добавить в `.env` как `VK_TOKEN=ваш_токен`

## `vk:token-get-group` - Получение community access token

Интерактивная команда для получения ключа доступа сообщества (community access token) через OAuth 2.0.

**Использование:**
```bash
# Интерактивный режим (команда запросит все параметры)
php artisan vk:token-get-group

# С указанием всех параметров
php artisan vk:token-get-group --client-id=12345678 --client-secret=секретный_ключ --redirect-uri=https://oauth.vk.com/blank.html --scopes=photos,messages
```

**Параметры:**
- `--client-id` - ID приложения VK (Application ID) - обязательный
- `--client-secret` - Секретный ключ приложения (Client Secret) - обязательный
- `--redirect-uri` - URL для перенаправления после авторизации (например, `https://oauth.vk.com/blank.html`) - обязательный
- `--scopes` - Права доступа через запятую (например: `photos,messages`)

**Поддерживаемые права доступа (scopes):**
- `photos` - Доступ к фотографиям (4)
- `app_widget` - Виджет приложения (64)
- `messages` - Доступ к сообщениям (4096)
- `docs` - Доступ к документам (131072)
- `manage` - Управление сообществом (262144)

**Как работает:**
1. Команда запрашивает параметры приложения (client_id, client_secret, redirect_uri, scopes)
2. Генерирует URL для авторизации
3. Показывает URL, который нужно открыть в браузере
4. После авторизации пользователь выбирает сообщества и вставляет URL с кодом
5. Команда получает токены для каждого выбранного сообщества
6. Выводит токены для добавления в `.env`

**Примечания:**
- Команда возвращает отдельный токен для каждого выбранного сообщества
- Формат токена в `.env`: `VK_TOKEN_GROUP_ID=токен_для_группы`
- Или можно использовать основной `VK_TOKEN` для первой группы

## `vk:token-check` - Проверка прав доступа токена

Проверяет валидность токена VK API и выводит информацию о правах доступа, которыми обладает токен.

**Использование:**
```bash
# Базовая проверка токена из .env
php artisan vk:token-check

# Проверка конкретного токена
php artisan vk:token-check --token=ваш_токен_здесь

# Проверка прав конкретного пользователя
php artisan vk:token-check --user-id=12345678

# Вывод в JSON формате
php artisan vk:token-check --format=json

# Проверка токена с выводом в JSON
php artisan vk:token-check --token=ваш_токен --format=json --output=token_info.json

# Подробный вывод с отладочной информацией
php artisan vk:token-check --verbose
```

**Параметры:**
- `--token` - Токен для проверки (опциональный, по умолчанию используется токен из `.env`)
- `--user-id` - ID пользователя для проверки прав (опциональный, по умолчанию проверяются права текущего пользователя)
- `--format=table` - Формат вывода: `table`, `json` (по умолчанию `table`)
- `--output` - Путь к файлу для сохранения результатов (опциональный)
- `--verbose` - Подробный вывод с отладочной информацией (битовая маска, процесс проверки)

**Что проверяется:**
- ✅ Валидность токена
- ✅ Информация о пользователе
- ✅ Права доступа:
  - `wall` - Доступ к стенам (чтение постов) - **обязательно**
  - `groups` - Доступ к группам - **обязательно**
  - `photos` - Доступ к фотографиям
  - `audio` - Доступ к аудиозаписям
  - `offline` - Бессрочный токен
  - `stats` - Доступ к статистике групп и приложений

**Примечания:**
- Команда использует современный метод `account.getAppPermissions` для получения битовой маски прав (быстро и эффективно)
- Для прав, которые могут не отображаться в битовой маске (например, `wall` для некоторых типов приложений), используется гибридный подход: проверка через битовую маску + дополнительная проверка через API
- Обязательные права (`wall`, `groups`) должны быть предоставлены токену для корректной работы большинства команд
- При указании опции `--token` токен временно подменяется в конфигурации, оригинальный токен восстанавливается после выполнения команды
